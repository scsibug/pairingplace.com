We are collecting short descriptions of programming tasks, to help us make recommendations for pairing programmers on open source projects. We would like your assistance, by describing real programming tasks. The following examples and guidelines should help show you what we're after.

Examples:

  • "Add friction, jitter, and oscillating gravity to a particle simulation iPhone application, in Objective-C."
  • "Create a feedparser-style library for Clojure, that can parse RSS and Atom feeds into native data structures."
  • "Support object versioning in hS3, the Haskell API to Amazon's Simple Storage Service (S3)."

Guidelines:

  • Succinct descriptions, under 140 characters is ideal.
  • Specific technologies or frameworks are listed.
  • Understandable with no more context than just the description.
  • Valuable to others (these tasks should not be solely about your own learning).
  • Small, something that could be reasonably achieved in ~4 hours by a couple motivated people.
  • Testable, it should be clear when the task is accomplished.
  • Real, from an existing open source project, either something you've wanted, done, or seen done. If at all possible, please cite the project, issue, or commit in the "Reference" field.

Please submit this form once for each task description you can come up with. Forward to any programmers you think want to help! Thank you.
